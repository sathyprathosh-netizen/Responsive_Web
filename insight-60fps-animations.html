60fps Animations on Mobile

Power Trio Editorial
8 min read

Mobile devices have caught up to desktops in raw power, but the animation gap persists. A smooth 60fps animation on your MacBook Pro will stutter at 24fps on a mid-range Android. Here's how we guarantee fluid motion everywhere.

The 16ms Budget

At 60fps, each frame has exactly 16.67 milliseconds to complete. That includes JavaScript execution, style calculations, layout, paint, and compositing. Miss that budget on even a single frame and users notice the jank.

Users can perceive animation stutter as low as 2 dropped frames. Your 58fps isn't "good enough" — it's broken.

The Compositor Thread

The single biggest performance win is moving animations off the main thread. Only two CSS properties are compositor-friendly: transform and opacity. Every other property triggers layout or paint, which blocks the main thread.

/* BAD — triggers layout */
.box { transition: width 0.3s, left 0.3s; }

/* GOOD — compositor only */
.box { transition: transform 0.3s, opacity 0.3s; }

/* Move instead of repositioning */
.box.moved { transform: translateX(200px); }

/* Scale instead of resizing */
.box.expanded { transform: scale(1.5); }

will-change: The Double-Edged Sword

The will-change property hints to the browser that an element will animate, allowing it to create a dedicated GPU layer. But over-use creates memory pressure and can actually hurt performance.

Our rules:

  • Apply will-change only to elements actively animating
  • Remove it after animation completes
  • Never apply it to more than 10 elements simultaneously
  • Use contain: layout style paint as a lighter alternative

requestAnimationFrame Done Right

Never use setTimeout or setInterval for animations. requestAnimationFrame synchronizes with the browser's paint cycle, giving you the full 16ms budget and automatically pausing in inactive tabs. 

function animate(timestamp) {
  const delta = timestamp - lastTime;
  lastTime = timestamp;
  position += velocity * (delta / 1000);
  element.style.transform = `translateX(${position}px)`;
  requestAnimationFrame(animate);
}
requestAnimationFrame(animate);

Testing on Real Devices

Chrome DevTools' performance profiler is a start, but it doesn't catch what you'll see on a $200 Android phone. We test every animation on at minimum: a flagship iOS device, a mid-range Android (Pixel 6a or equivalent), and a budget device (sub-$200). If it's smooth on the budget device, it's smooth everywhere.
