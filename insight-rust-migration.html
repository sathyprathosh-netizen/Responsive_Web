Why We Switched to Rust

Power Trio Editorial
9 min read

In early 2025, we made a decision that terrified half our engineering team: we migrated our core backend services from Node.js to Rust. A year later, we can say without hesitation — it was the best technical decision we've ever made.

Why Node.js Wasn't Enough

Let's be clear: Node.js is a fantastic runtime. We built our first 15 projects on it. But as our client workloads grew — particularly in fintech and healthcare — we kept hitting the same walls:

  • Memory usage spiking unpredictably under load
  • Garbage collection pauses causing latency spikes
  • No compile-time guarantees for critical data transformations
  • Difficulty reasoning about concurrency in complex async workflows

We didn't leave Node.js because it was bad. We left because our problems outgrew its comfort zone.

The Benchmarks

We ran identical workloads — a real-world API serving JSON with database queries, authentication, and rate limiting — on both stacks:

Metric Node.js (Express) Rust (Axum) Improvement
Requests/sec 12,400 89,000 7.2×
P99 Latency 145ms 8ms 18×
Memory (idle) 120MB 4MB 30×
Memory (peak) 850MB 28MB 30×
Cold start 1.2s 12ms 100×

The Learning Curve

Yes, Rust is harder to learn than JavaScript. The borrow checker will humble you. Lifetimes will confuse you. But here's the thing: Rust's compiler errors are the best documentation you'll ever read.

Our team's onboarding timeline:

  • Week 1-2: Fighting the borrow checker constantly
  • Week 3-4: Understanding ownership, writing basic services
  • Month 2: Productive. Shipping features at 80% of Node.js speed
  • Month 3+: Faster than Node.js. Fewer bugs. Better architecture

What We Still Use Node.js For

Not everything needs Rust. We still use Node.js for:

  • Rapid prototyping and proof-of-concepts
  • Simple CRUD APIs with low traffic
  • Build tooling and developer scripts
  • SSR with Next.js (React's ecosystem is unmatched)

The key is choosing the right tool for the job. For performance-critical, long-running services that handle sensitive data — Rust. For everything else — whatever ships fastest.

Rust didn't replace our stack. It completed it.
