Dark Mode Done Right

Power Trio Editorial
6 min read

Dark mode isn't just "invert everything." Done poorly, it causes eye strain, hides content, and breaks brand identity. Done right, it becomes a feature users love and actively choose. Here's how we approach dark themes across all our projects.

Why Not Just Invert?

The naive approach — filter: invert(1) — flips every color. Images become negatives. Carefully chosen brand colors get destroyed. Shadows that made sense on light backgrounds look wrong on dark ones.

True dark mode requires rethinking your entire color system from the ground up, not layering a filter on top of an existing design.

Dark mode is a separate design language, not a CSS toggle.

The Color Architecture

Our dark mode system uses three layers of CSS custom properties:

:root {
  /* Raw palette — never used directly */
  --gray-50: #F8FAFC;
  --gray-900: #0F172A;
  --indigo-500: #6366F1;

  /* Semantic tokens — the bridge */
  --color-bg-primary: var(--gray-50);
  --color-text-primary: var(--gray-900);
  --color-accent: var(--indigo-500);
}

[data-theme="dark"] {
  /* Only semantic tokens change */
  --color-bg-primary: var(--gray-900);
  --color-text-primary: var(--gray-50);
  --color-accent: var(--indigo-400);
}

Components reference only semantic tokens, never raw palette values. This means switching themes requires zero component changes.

Elevation, Not Lightness

In light mode, higher-elevation surfaces are lighter (white cards on gray backgrounds). In dark mode, the opposite is true — higher-elevation surfaces are lighter shades of dark. Material Design calls this "elevation overlay."

Each elevation step adds approximately 5% white overlay to the base background. This creates a natural visual hierarchy without using shadows (which are barely visible on dark backgrounds).

Contrast Ratios

WCAG requires a minimum contrast ratio of 4.5:1 for normal text and 3:1 for large text. In dark mode, this is harder to achieve because our instinct is to use full white (#FFFFFF) text — but that creates excessive contrast that causes eye strain.

Our rules:

  • Primary text: #F1F5F9 (not pure white) — contrast 14:1
  • Secondary text: #94A3B8 — contrast 5.5:1
  • Tertiary/disabled: #64748B — contrast 3.2:1 (large text only)
  • Avoid pure white: #FFFFFF is too harsh on dark backgrounds

Respecting User Preference

Always default to the system preference using prefers-color-scheme, then allow manual override with a toggle that persists via localStorage:

const getTheme = () => {
  const stored = localStorage.getItem('theme');
  if (stored) return stored;
  return window.matchMedia(
    '(prefers-color-scheme: dark)'
  ).matches ? 'dark' : 'light';
};

document.documentElement.setAttribute(
  'data-theme', getTheme()
);

The critical detail: this script must run in the <head> before any content renders to prevent a "flash of wrong theme" (FOWT).

The best dark mode experience is one the user never has to think about.
